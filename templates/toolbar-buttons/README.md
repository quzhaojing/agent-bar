# Toolbar 按钮配置模板

这个目录包含了 Agent Bar 的预定义工具栏按钮配置模板，可以帮助用户快速设置不同场景下的工具按钮。

## 模板分类

### 📝 通用文本处理 (`general.json`)
适用于日常文本处理的基本工具按钮
- **解释**: 用简单易懂的语言解释内容
- **总结**: 将内容总结为要点
- **翻译**: 翻译成中文
- **简化**: 使内容更容易理解
- **扩展**: 对内容进行详细扩展和说明

**适用场景**: 日常网页浏览、文档阅读、信息获取
**标签**: `text`, `general`, `daily`

### 💻 编程开发 (`coding.json`)
适用于程序员和代码开发的工具按钮
- **解释代码**: 解释代码的功能和工作原理
- **优化代码**: 提高性能和可读性
- **调试代码**: 分析代码中的错误和问题
- **重构代码**: 使代码更加模块化和可维护
- **添加注释**: 添加详细的注释和文档
- **转换语言**: 转换成其他编程语言
- **生成测试**: 生成单元测试

**适用场景**: GitHub、Stack Overflow、GitLab 等开发平台
**标签**: `code`, `development`, `programming`, `debug`

### ✍️ 写作助手 (`writing.json`)
适用于写作和内容创作的工具按钮
- **改进写作**: 改进语法、词汇和表达
- **检查语法**: 检查语法错误并提供修改建议
- **改变语气**: 改变文本的语气
- **精简文本**: 保留核心信息，去除冗余内容
- **扩展内容**: 增加更多细节和例子
- **重写风格**: 用不同的风格重新表达

**适用场景**: Notion、Google Docs、Medium、WordPress 等写作平台
**标签**: `writing`, `content`, `creative`, `grammar`

### 🧠 学习研究 (`learning.json`)
适用于学习和学术研究的工具按钮
- **解释概念**: 详细解释概念的定义、原理和应用
- **举例说明**: 提供具体的例子和应用场景
- **创建大纲**: 基于内容创建学习大纲
- **生成测验**: 生成练习题和测验
- **概念对比**: 分析概念的相同点和不同点
- **历史背景**: 提供概念的历史背景和发展过程

**适用场景**: Wikipedia、Coursera、edX、可汗学院、YouTube 教育频道
**标签**: `learning`, `education`, `study`, `research`, `academic`

### 💼 商务办公 (`business.json`)
适用于商务和办公场景的工具按钮
- **撰写邮件**: 基于要点撰写专业的商务邮件
- **创建摘要**: 创建简洁的商务摘要
- **生成报告**: 整理成结构化的报告
- **头脑风暴**: 提供创新的想法和建议
- **SWOT分析**: 进行优势、劣势、机会、威胁分析
- **创建提案**: 创建商务提案

**适用场景**: LinkedIn、Slack、Microsoft Office、Google Workspace、Notion
**标签**: `business`, `office`, `professional`, `corporate`, `work`

## 如何使用这些模板

### 方法 1: 在设置页面中导入
1. 打开 Agent Bar 设置页面
2. 在"Toolbar Buttons"标签下
3. 点击"Import Template"按钮
4. 选择相应的模板文件

### 方法 2: 手动配置
1. 打开模板文件查看配置
2. 在设置页面中手动添加对应的按钮
3. 复制模板中的 `promptTemplate` 和其他配置

### 方法 3: 直接编辑配置文件
1. 导出当前配置
2. 将模板中的按钮配置合并到导出的配置文件
3. 导入修改后的配置文件

## 自定义模板

你可以创建自己的模板文件，格式如下：

```json
{
  "name": "模板名称",
  "description": "模板描述",
  "category": "分类",
  "buttons": [
    {
      "id": "unique-id",
      "name": "按钮显示名称",
      "icon": "emoji-icon",
      "promptTemplate": "提示词模板，使用 {{selectedText}} 作为变量",
      "order": 1,
      "enabled": true
    }
  ],
  "urlRules": ["适用的域名规则"],
  "tags": ["标签1", "标签2"]
}
```

## 注意事项

1. **ID 唯一性**: 每个 `id` 都应该是唯一的
2. **提示词变量**: 使用 `{{selectedText}}` 作为选中文本的占位符
3. **URL 规则**: `urlRules` 数组定义了在哪些网站显示这些按钮
4. **图标**: 推荐使用 Emoji 作为图标，简洁直观
5. **排序**: `order` 数字越小，按钮在工具栏中的位置越靠前
